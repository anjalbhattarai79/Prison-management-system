## Note on Usage

- **Always check your coursework guidelines** for the required citation format (IEEE, APA, Harvard, etc.)
- **All sources listed are academically acceptable** - peer-reviewed journals, university press textbooks, and official documentation
- **Cormen et al. [1] is the gold standard** for data structures and algorithms - cite it for core concepts
- **Knuth [2] is highly respected** for in-depth algorithm analysis and historical context
- **Use recent editions** where possible to show current understanding
- **Official documentation (Oracle)** is fully acceptable for technical implementation details

### When to Cite:
- **Always cite when you**:
  - Define technical terms (LinkedList, Queue, Stack, binary search, LIFO, FIFO, etc.)
  - Discuss algorithm complexity (O(n), O(log n), O(1))
  - Reference established design patterns (MVC, Separation of Concerns)
  - Mention best practices or industry standards
  - Discuss theoretical foundations of your implementation
  - Compare different approaches or data structures

### In-Text Citation Examples:

**IEEE Style:**
```
"The LinkedList data structure provides efficient insertion and deletion operations [1]."
"Binary search achieves O(log n) time complexity in sorted arrays [8]."
"The MVC pattern separates presentation from business logic [10]."
```

**APA Style:**
```
"The LinkedList data structure provides efficient insertion and deletion operations (Cormen et al., 2009)."
"Binary search achieves O(log n) time complexity in sorted arrays (Sedgewick & Wayne, 2011)."
"The MVC pattern separates presentation from business logic (Gamma et al., 1994)."
```
